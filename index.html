Harness Engineering: The Real Skill

2025 was agents. 2026 is agent harnesses.

What is a harness?

The system that wraps around an AI model to manage long-running tasks reliably. The model generates responses. The harness handles everything else.

  • Permissions & sandboxing
  • State persistence across sessions
  • Tool orchestration & MCP servers
  • Lifecycle management (launchd, containers)
  • Error recovery & graceful degradation
  • Memory & context management

NanoClaw Architecture

Here's the full architecture. Host process handles messaging and routing. Each group gets its own Linux container via Apple Container. Main group has full project access, other groups are sandboxed to their own directory. Key insight: security is structural, not just code-level. A compromised family-chat agent literally cannot see other groups' data.

Here's the security comparison. OpenClaw does application-level sandboxing in a shared process — if one adapter is compromised, everything is. NanoClaw uses OS-level container isolation. The mount allowlist lives OUTSIDE the project directory so containers can't modify their own security config. These blocked patterns prevent accidental credential exposure.

The Code: Container Isolation

container-runner.ts — How agents are spawned

// Each group gets isolated volume mounts
function buildVolumeMounts(group: RegisteredGroup, isMain: boolean): VolumeMount[] {
  const mounts: VolumeMount[] = [];

  if (isMain) {
    mounts.push({ hostPath: projectRoot,
                   containerPath: '/workspace/project', readonly: false });
    // SSH keys for git push (main only, read-only)
    mounts.push({ hostPath: sshDir,
                   containerPath: '/home/node/.ssh', readonly: true });
  } else {
    // Non-main groups: ONLY their own folder
    mounts.push({ hostPath: path.join(GROUPS_DIR, group.folder),
                   containerPath: '/workspace/group', readonly: false });
    // Global memory (read-only)
    mounts.push({ hostPath: globalDir,
                   containerPath: '/workspace/global', readonly: true });
  }

  // Per-group isolated sessions (prevents cross-group session access)
  mounts.push({ hostPath: groupSessionsDir,
                 containerPath: '/home/node/.claude', readonly: false });

  // Per-group IPC namespace (prevents cross-group privilege escalation)
  mounts.push({ hostPath: groupIpcDir,
                 containerPath: '/workspace/ipc', readonly: false });

  // Only expose auth vars, not entire .env
  const allowedVars = ['CLAUDE_CODE_OAUTH_TOKEN', 'ANTHROPIC_API_KEY'];
  return mounts;
}

// Spawn the container
const container = spawn('container', containerArgs, { stdio: ['pipe', 'pipe', 'pipe'] });
container.stdin.write(JSON.stringify(input));
container.stdin.end();
This is the actual code. Each group gets its own mount namespace. Non-main groups cannot see the project root. Sessions are isolated per group. IPC directories are namespaced to prevent cross-group privilege escalation. The container is ephemeral — fresh environment per invocation.

Oil & Gas / Energy

  • Pipeline monitoring: pressure sensors trigger event → agent analyzes pattern → autonomous isolation OR human escalation
  • Electricity grid: frequency deviation → agent checks load balance → adjusts demand response OR pages operator
  • Well production: decline curve anomaly → agent pulls offset well data → generates recommendation

The key shift: These workflows were once monitored by humans making heuristic-based decisions. Event-driven agents replace heuristics with contextual reasoning — and escalate when uncertain.

Here's where it gets interesting for business. The exact same event-driven pattern I use at home maps directly to industrial applications. WhatsApp message becomes SCADA event. Kitchen pantry becomes pipeline pressure. The architecture is identical: event triggers agent, agent reasons contextually, agent acts or escalates. In Oil & Gas, this replaces human operators making heuristic decisions — if pressure drops 10%, shut down. Instead: agent analyzes the full context, checks historical data, makes a nuanced decision, and only escalates when genuinely uncertain.

Serverless Event-Driven Agents

The architecture scales from Mac to cloud

Scale this up. Events from SCADA, grid sensors, well data flow into an event bus. Serverless agent harnesses spin up per event — just like NanoClaw spawns a container per message. The agent loads context, reasons with MCP tools (pulling historical data, checking thresholds, consulting runbooks), and then either acts autonomously, logs the decision, notifies, or escalates to a human. This replaces heuristic-based automation with contextual reasoning.
