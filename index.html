The question: how much code do you need for an always-on personal AI agent?

Claude Code shipped with a crab mascot. Community ran with it. Peter Steinberger built OpenClaw — 400K lines connecting every messaging app. NanoClaw asked: what if 500 lines? The real question is architectural.

OpenClaw vs NanoClaw

  • 🔴 Shared Node.js process
  • 🔴 CVE-2026-25253 (CVSS 8.8)

vs

NanoClaw

  • ✅ ~500 lines core / 6 deps
  • ✅ OS-level container isolation
  • ✅ Separate Linux VM per group
  • ✅ Auditable in 8 minutes

NanoClaw: "Don't add features. Add skills." — Channels (Telegram, Slack, Discord) added via /add-telegram skills that transform your fork. Core stays tiny.

Side-by-side. OpenClaw: 400K lines, shared process, one compromise exposes everything. NanoClaw: real container isolation. Skills philosophy is key: /add-telegram transforms your fork instead of bloating the core. The core stays small and auditable.

Harness Engineering

2025 was agents. 2026 is agent harnesses. — source

What is a harness?

The system wrapping the AI model for reliable long-running tasks. (Anthropic blog)

  • Permissions & sandboxing
  • State persistence across sessions
  • Tool orchestration (MCP servers)
  • Lifecycle (launchd, containers)
  • Context engineering — right context, right tools, right time
"I haven't written a single line of code myself since December 2025. I run 10-20 AI agents in parallel." — Andrej Karpathy, March 2026

The pattern: tiny core, extensible via skills, iterate — add context, tools, and autonomy gradually.

Karpathy's AutoResearch: 630 lines. NanoClaw: ~500 lines. Same insight.

Anthropic published guidance on harnesses for long-running agents. The harness manages permissions, state, tools, lifecycle. Karpathy's AutoResearch: 630 lines. NanoClaw: 500. Key: don't build a big app. Build a tiny harness and iteratively add skills, context, tools. Context engineering is the real skill.

NanoClaw Architecture

Walk the diagram. Host: single Node process with Telegram, SQLite, scheduler. Each group gets its own container — a real Linux VM. Containers are EPHEMERAL: spawned per message, destroyed with --rm. Zero idle cost. Docker partnership enables cross-platform — Apple Container on macOS, Docker on Linux. Security is structural: a compromised family-chat agent literally cannot see other groups' data.

container-runner.ts

if (isMain) {
  mounts.push({ hostPath: projectRoot,
    containerPath: '/workspace/project' });
} else {
  // Non-main: ONLY their own folder
  mounts.push({ hostPath: groupDir,
    containerPath: '/workspace/group' });
}
// Per-group isolated sessions
mounts.push({ hostPath: groupSessionsDir,
  containerPath: '/home/node/.claude' });

// Only 2 env vars exposed
const allowedVars = [
  'CLAUDE_CODE_OAUTH_TOKEN',
  'ANTHROPIC_API_KEY'
];
Security comparison. OpenClaw: shared process, app-level sandbox. NanoClaw: OS containers, separate VMs. Mount allowlist at ~/.config/nanoclaw/ — outside the project, containers can't modify their own security. Only 2 env vars exposed. This is the actual code from container-runner.ts.

The iterative approach

  • Step 1: Deploy the harness (500 lines)
  • Step 2: Add skills (/add-telegram)
  • Step 3: Add MCP tools (kitchen, calendar)
  • Step 4: Add event triggers (cron, RSS feeds, webhooks)
  • Step 5: Refine context engineering (CLAUDE.md)

Each step adds capability. The cognitive layer builds up gradually.

Near-autonomous on the spectrum. The key: iterate. Start with a tiny harness, add skills, add MCP tools, add event triggers like cron, RSS feeds, webhooks. Each step adds capability. Context engineering — refining CLAUDE.md files, available tools, scheduled behaviors — is the ongoing work. You don't build everything at once.

Same event-driven pattern maps to industrial applications. Pipeline monitoring: pressure event triggers agent, agent loads historical data via MCP tools, analyzes context, either acts autonomously or escalates. Electricity grid: frequency deviation triggers the same pattern. The key shift: humans used heuristic rules — if pressure drops 10%, shut down. Agents do contextual reasoning — check history, correlations, runbooks — and escalate when genuinely uncertain.
